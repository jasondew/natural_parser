SUTF8 - UTF-8 Format for Sources

This Natural profile parameter specifies the default format to be used when Natural sources are saved.

It is valid only for Natural Development Server sessions; it will be ignored in a native Natural for UNIX session.

See also Profile Parameters in the Unicode and Code Page Support documentation.

Possible settings ON

The default format for saving Natural sources is UTF-8.

All sources will be saved in UTF-8 format, which assures that the source content does not depend on the installed system code page.

If the profile parameter SRETAIN is also set to ON, only newly created sources will be saved in UTF-8 format. Existing sources will then be saved in the original encoding, if possible.

OFF

The default format for saving Natural sources is "code page".
